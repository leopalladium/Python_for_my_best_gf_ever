## 🔑 Критические разработческие паттерны

### 1. **Типизация везде** (Type Hints)
Код использует аннотации типов в КАЖДОЙ функции:
```python
def multiply_numbers(bound: int, numbers: List[int]) -> int:
```
**При добавлении кода:** всегда включай типы параметров и возвращаемого значения.

### 2. **Временная и пространственная сложность обязательна**
Каждая функция должна иметь в docstring:
```python
"""
...описание...
временная сложность: O(...)
пространственная сложность: O(...)
"""
```
**При изменении функций:** пересчитай сложность и обнови docstring!

### 3. **PEP8 + дополнительные требования**
- Строки < 80 символов
- Два пустых бланка между функциями
- Англоязычные названия переменных
- Проверка: `pycodestyle имя_файла.py`

### 4. **Работа с граничными случаями**
Многие ошибки в коде связаны с краевыми случаями:
- Индексация с нуля (Python) — **не забывай проверять `i==0`**
- Выход за границы массива — **всегда добавляй условия вроде `if 1 <= numbers[i] <= bound`**
- Бесконечные циклы из-за неправильных отступов — **тщательно проверяй уровень отступа**

### 5. **Без встроенных функций для строк**
В части II домашнего задания **запрещено** использовать встроенные методы строк (`.reverse()`, `.find()`, `.split()` и т.д.). Требуется ручная реализация.

## 🧪 Тестирование

### Запуск тестов
```bash
uv run python "ukol_1_zadani (1).py"
```

### Структура тестов
Тесты встроены в конец `.py` файла и вызываются автоматически:
- `ib114_test_palindrom()` — проверка палиндромов
- `ib114_test_list()` — работа со связными списками (5 тестов)
- `ib114_test_multiply_numbers()` — произведение чисел в диапазоне
- `ib114_test_brackets()` — правильность скобок
- `ib114_test_sequence_sum()` — специальная сумма последовательности
- `ib114_test_find()` — поиск подстроки

**Важно:** Все тесты должны выводить "OK." для успеха.

### Распространённые ошибки в тестах

1. **IndexError при доступе к массиву** — обычно из-за неправильной индексации
2. **AttributeError при работе со связными списками** — неправильная работа с `None`
3. **Logic errors** — функция выполняется, но результат неправильный
